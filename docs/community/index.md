## Contributing

We welcome contributions to any of our packages. Each repository has its own contributing guide — a good place to start is the `CONTRIBUTING.md` file in the relevant repo, or the documentation site for that package.

### General guidelines

- **Bug reports** — Open an issue in the relevant GitHub repository with a minimal reproducible example.
- **Feature requests** — Open an issue describing the use case and expected behaviour before starting implementation.
- **Pull requests** — Fork the repository, create a branch, and open a PR against `master`. All PRs run the full test suite via GitHub Actions.
- **Documentation** — Documentation improvements are always appreciated. Each package uses [Zensical](https://zensical.org) for its docs, with source files in the `docs/` directory.

### Code quality

All packages enforce:

- [Ruff](https://docs.astral.sh/ruff/) for linting and formatting
- [pytest](https://pytest.org) with coverage reporting via [codecov](https://codecov.io)
- Static analysis via [SonarCloud](https://sonarcloud.io) (selected packages)
